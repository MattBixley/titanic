A penalised linear baseline with glmnet — interpretable and fast

Where we are: A fourth base learner for the Titanic — Tidymodels Tour, following the same recipe → spec → workflow → tune → finalize → evaluate → save structure as the XGBoost, Random Forest and Neural Net docs. It also feeds the stacked ensemble.

Why logistic regression?

It is the classic starting point for binary classification: a linear model of the log-odds of survival. With the glmnet engine we get elastic-net regularisation — penalty (how much to shrink) and mixture (ridge ↔︎ lasso) — which guards against overfitting the wide, PCA-augmented feature space and does automatic feature selection. It is fast, interpretable (signed coefficients), and a strong, honest baseline the fancier models must beat.

Coefficients (the linear story)

Unlike the tree models, a logistic regression hands us signed effects: the log-odds contribution of each (standardised) feature at the chosen penalty. Positive pushes toward survival, negative toward not.

Show code 
logistic_fit |>
  extract_fit_parsnip() |>
  tidy() |>
  filter(term != "(Intercept)", estimate != 0) |>
  slice_max(abs(estimate), n = 15) |>
  ggplot(aes(x = reorder(term, abs(estimate)), y = estimate,
             fill = estimate > 0)) +
  geom_col() +
  coord_flip() +
  scale_fill_manual(values = c(`TRUE` = kakapo_pal[1], `FALSE` = kakapo_pal[5]),
                    labels = c(`TRUE` = "toward survival", `FALSE` = "toward death"),
                    name = NULL) +
  labs(title = "Logistic Regression coefficients",
       x = NULL, y = "Coefficient (log-odds)") +
  theme_minimal()

The same engineered signals — GroupSurvival, Title, Sex, class — carry the weight, now with an explicit direction.

## Tuning with cross-validation

### Coefficients (the linear story)

Unlike the tree models, a logistic regression hands us **signed** effects: the
log-odds contribution of each (standardised) feature at the chosen penalty.
Positive pushes toward survival, negative toward not.
