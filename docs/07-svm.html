Why an SVM?

A support vector machine with a radial basis function (RBF) kernel draws a smooth, curved decision boundary in a high-dimensional feature space — a genuinely different shape of model from the trees, the linear logistic fit, and the small neural net. That difference is exactly what a stack likes: when the SVM is wrong, it tends to be wrong on different passengers. We tune cost (margin softness) and rbf_sigma (kernel width).

## Why an SVM?

A support vector machine with a **radial basis function (RBF)** kernel draws a
smooth, curved decision boundary in a high-dimensional feature space — a
genuinely different shape of model from the trees, the linear logistic fit, and
the small neural net. That difference is exactly what a stack likes: when the
SVM is wrong, it tends to be wrong on *different* passengers. We tune `cost`
(margin softness) and `rbf_sigma` (kernel width).

## The workflow

```{r}
#| label: workflow
svm_wf <- workflow() |>
  add_recipe(svm_recipe) |>
```

## Tuning with cross-validation

```{r}
#| label: resamples
set.seed(42)
folds <- vfold_cv(train, v = 10, strata = Survived)
```

```{r}
#| label: grid
preds <- prep(make_recipe(train, num_comp = 2)) |>
  bake(new_data = NULL) |>
  select(-Survived, -PassengerId)

svm_params <- extract_parameter_set_dials(svm_wf) |>
  finalize(preds) |>
  update(num_comp = num_comp(range = c(2L, 10L)))

set.seed(42)
svm_grid <- grid_space_filling(svm_params, size = 18)
```

```{r}
#| label: tune
#| message: false
#| warning: false

svm_metrics <- metric_set(roc_auc, accuracy)

set.seed(42)
svm_tuned <- tune_grid(
  svm_wf,
  resamples = folds,
  grid      = svm_grid,
  metrics   = svm_metrics,
  control   = control_grid(save_pred = TRUE, save_workflow = TRUE, event_level = "second")
)
```

```{r}
#| label: save-res
saveRDS(svm_tuned, here("models", "svm_res.rds"))
```

## Finalize and fit on the full training set

```{r}
best_svm
```

::: {.callout-note}
## No native variable importance
An RBF-kernel SVM is effectively a black box — there is no coefficient or split
to read importance from. If you want it, use a *model-agnostic* permutation
approach (e.g. `DALEX`/`vip::vi(method = "permute")`). We lean on the other
models for the importance story.
:::
