Model 2 — Random Forest

Bagged trees with ranger — a robust, low-tuning baseline

Where we are: Second of three model documents in the Titanic — Tidymodels Tour. It follows the same structure as the XGBoost doc — recipe → spec → workflow → tune → finalize → evaluate → save — so skim for the differences. Features come from the EDA via the shared make_recipe().

Why a random forest?

Where XGBoost grows trees sequentially to correct residuals, a random forest grows many independent deep trees on bootstrap samples and averages them. It is famously forgiving: with trees = 1000 fixed, there are really only two knobs worth tuning (mtry, min_n), it rarely overfits badly, and it gives a strong baseline with almost no fuss. We use the fast ranger engine and ask it for permutation importance.

Setup

source(here::here("R", "00_setup.R"))      # tidyverse + tidymodels, seed = 42
source(here::here("R", "02_features.R"))   # make_recipe(data)

The model specification

trees is fixed at 1000 (more is rarely worth it); we tune mtry and min_n. We set importance = "permutation" so we can plot variable importance later.

rf_spec <- rand_forest(
  mtry  = tune(),
  min_n = tune(),
  trees = 1000
) |>
  set_engine("ranger", importance = "permutation") |>
  set_mode("classification")

The workflow

rf_wf <- workflow() |>
  add_recipe(rf_recipe) |>
  add_model(rf_spec)

Tuning with 5-fold cross-validation

set.seed(42)
folds <- vfold_cv(train, v = 5, strata = Survived)
# mtry's upper bound depends on the number of predictors AFTER the recipe runs,
# so we finalize() its range against the prepped predictor set rather than
# guessing. min_n we give a plain range.
rf_params <- rf_wf |>
  extract_parameter_set_dials() |>
  finalize(x = make_recipe(train) |> prep() |> bake(new_data = NULL) |>
             select(-Survived))

set.seed(42)
rf_grid <- grid_latin_hypercube(rf_params, size = 20)
Warning: `grid_latin_hypercube()` was deprecated in dials 1.3.0.
ℹ Please use `grid_space_filling()` instead.
rf_metrics <- metric_set(roc_auc, accuracy)

set.seed(42)
rf_tuned <- tune_grid(
  rf_wf,
  resamples = folds,
  grid      = rf_grid,
  metrics   = rf_metrics,
  # Positive class "1" is the second factor level.
  control   = control_grid(save_pred = TRUE, event_level = "second")
)

## Why a random forest?

grows many *independent* deep trees on bootstrap samples and averages them. It
is famously forgiving: with `trees = 1000` fixed, there are really only two
