Building a Kaggle Submission

One parameterized document, three models, leaderboard-ready CSVs

Where we are: The final stop on the Titanic — Tidymodels Tour. Each model document (XGBoost, Random Forest, Neural Net) fits a workflow and saves it to models/<model>_fit.rds. This document is parameterized: tell it which model, and it loads that fit, predicts on the test set, and writes a Kaggle-ready CSV.

How to render this document per model

This .qmd has a model parameter (default "xgboost"). Override it on the command line with -P model:<value>:

# XGBoost (also the default if you omit -P)
quarto render docs/submission.qmd -P model:xgboost

# Random forest
quarto render docs/submission.qmd -P model:rf

# Neural net
quarto render docs/submission.qmd -P model:nnet

Each render reads the matching models/<model>_fit.rds, writes models/submission_<model>.csv, and prints the exact upload command. You must have fitted (rendered) the corresponding model document first so the .rds exists.

Setup

# We need the tidymodels stack to load the saved workflow and call predict().
# 00_setup.R also sets paths and the seed.
source(here::here("R", "00_setup.R"))
library(readr)

Resolve and validate the requested model

# The parameter comes in from the YAML default or the -P flag on the CLI.
requested_model <- params$model

# Map each valid model key to its saved fit. This single lookup table is the
# source of truth for which models are submittable.
model_paths <- c(
  xgboost = here::here("models", "xgboost_fit.rds"),
  rf      = here::here("models", "rf_fit.rds"),
  nnet    = here::here("models", "nnet_fit.rds")
)

# Fail loudly and helpfully if someone passes an unknown model.
if (!requested_model %in% names(model_paths)) {
  stop(
    "Unknown model '", requested_model, "'. ",
    "Choose one of: ", paste(names(model_paths), collapse = ", "), ". ",
    "Example: quarto render docs/submission.qmd -P model:rf",
    call. = FALSE
  )
}

fit_path <- model_paths[[requested_model]]

# Make sure the fitted workflow actually exists — i.e. the model doc was
# rendered first.
if (!file.exists(fit_path)) {
  stop(
    "Fitted model not found at '", fit_path, "'. ",
    "Render the matching model document first (e.g. quarto render ",
    "docs/02-xgboost.qmd) so the .rds is saved.",
    call. = FALSE
  )
}

message("Building submission for model: ", requested_model)

Predict and assemble the submission

# predict() runs the workflow's recipe on the raw test data, then the model.
# type = "class" returns the hard 0/1 label Kaggle wants.
preds <- predict(model_fit, new_data = test, type = "class")

# Kaggle requires EXACTLY two columns: PassengerId and Survived, with Survived
# as 0/1 integers. .pred_class is a factor ("0"/"1"); convert via character so
# we don't accidentally get factor codes (1/2).
submission <- tibble::tibble(
  PassengerId = test$PassengerId,
  Survived    = as.integer(as.character(preds$.pred_class))
)

# Sanity checks: right number of rows, right value set, no missing predictions.
stopifnot(
  nrow(submission) == nrow(test),
  all(submission$Survived %in% c(0L, 1L)),
  !anyNA(submission$Survived)
)

head(submission, 10)

out_path <- here::here("models", paste0("submission_", requested_model, ".csv"))

readr::write_csv(submission, out_path)

Submit to Kaggle

Push the CSV to the leaderboard with the Kaggle CLI. The command below is built for the model you rendered:

kaggle competitions submit -c titanic -f models/submission_xgboost.csv -m "xgboost via tidymodels"
First time? Authenticate first

The kaggle CLI needs your API token (~/.kaggle/kaggle.json) before any submit or download will work. The Kaggle setup guide walks through creating and installing it.

Daily submission limit

Kaggle caps Titanic submissions per day (commonly ~10). Submit your best candidate by CV score rather than spraying every experiment at the leaderboard.

## Submit to Kaggle

Push the CSV to the leaderboard with the Kaggle CLI. The command below is built
for the model you rendered:

```{r}
#| label: kaggle-cmd
#| echo: false
#| results: asis

cmd <- sprintf(
  'kaggle competitions submit -c titanic -f models/submission_%s.csv -m "%s via tidymodels"',
  requested_model, requested_model
)
cat("```bash\n", cmd, "\n```\n", sep = "")
```
